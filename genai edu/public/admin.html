Prompt Cues (case-specific)
Optional. If set, these override the global cues when students open this case. Leave empty to use global cues.
Roles
Optional. Each role has a label (e.g. CTO, Manager) and a brief — what this role knows, wants, or is constrained by. When students open this case, they'll be asked to pick one before chatting. Leave empty if the case doesn't have distinct sides.

Settings

Brand, AI behavior, prompt cues, research stages. All changes take effect on the next student login or next message.

Brand & Identity

What the student sees on the login page and sidebar.

Short name
Sidebar label and tab title. E.g. GRA6842 or Pilot Study A.
Display title
Two-word title shown on the login screen (last word will be italicized). E.g. Mastering Negotiation.
Tagline
One-line description under the login title.
AI name
What students see as the AI's identity. E.g. Digital Robin, Mentor, Apprentice.

Welcome & Behavior

First impression and AI ground rules.

Welcome message
First AI message every student sees. Use {name} to insert the student's first name. Markdown (**bold**, *italic*) and emoji work.
AI system prompt
Defines how the AI behaves — its role, tone, what concepts to emphasize, how to respond. Stage-specific instructions (below) are appended on top of this.

Prompt Cues

Clickable suggestions shown above the student's chat input. Up to 20.

Research Stages

Predefined phases. Students can switch between them; each stage's instruction is added to the AI's system prompt when that stage is active. The default stage is always present.

Tips
  • System prompt: be specific about role, tone, and what NOT to do. List domain concepts you want referenced.
  • Prompt cues: short labels, full questions. A good cue starts a productive conversation.
  • Test after saving: open the student view in an incognito window with a test name.
